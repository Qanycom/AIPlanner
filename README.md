# Intelligent Task Planner

**Intelligent Task Planner** — вебзастосунок для планування задач з адаптивними рекомендаціями, пріоритизацією, календарем та нагадуваннями.  
Проєкт розроблено як дипломну роботу на тему: **«Розробка інтелектуального планувальника задач з адаптивними рекомендаціями»**.

## Опис проєкту

Застосунок допомагає користувачу організовувати особисті задачі, визначати їхню важливість, отримувати рекомендації щодо наступної задачі та не пропускати дедлайни завдяки системі нагадувань.

Основна ідея проєкту — не просто зберігати список задач, а допомагати користувачу приймати рішення, яку задачу краще виконати першою.

## Основні можливості

- додавання, редагування та видалення задач;
- встановлення пріоритету задачі: низький, середній, високий;
- вибір дедлайну, дати та часу виконання;
- зміна статусу задачі:
  - `todo`;
  - `in progress`;
  - `done`;
- автоматичне формування рекомендованої задачі;
- розрахунок важливості задачі за пріоритетом, дедлайном і кількістю відкладень;
- календар із відображенням задач по днях;
- архів виконаних задач;
- нагадування про задачі через Notification API;
- адаптивний інтерфейс для комп’ютера та мобільних пристроїв;
- можливість використання як PWA-застосунок;
- реєстрація, авторизація та підтвердження пошти;
- відновлення пароля через email;
- AI-поради щодо виконання задач.

## Логіка пріоритезації задач

Для визначення рекомендованої задачі використовується оцінка важливості:

```text
score = priorityScore + deadlineScore + postponedCount * 2
```

Де:

- `priorityScore`:
  - high = 3;
  - medium = 2;
  - low = 1;

- `deadlineScore`:
  - прострочена задача = +5;
  - дедлайн протягом 1 дня = +3;
  - дедлайн протягом 3 днів = +2;
  - інші задачі = +1;

- `postponedCount` — кількість відкладень задачі.

Якщо декілька задач мають однаковий бал, перевага надається задачі з ближчою датою та часом виконання.

## Встановлення та запуск

### 1. Клонування репозиторію

```bash
git clone https://github.com/username/intelligent-task-planner.git
cd intelligent-task-planner
```

### 2. Запуск backend

Перейдіть у папку backend:

```bash
cd backend
```

Створіть віртуальне середовище:

```bash
python -m venv venv
```

Активуйте його:

#### Windows

```bash
venv\Scripts\activate
```

#### macOS / Linux

```bash
source venv/bin/activate
```

Встановіть залежності:

```bash
pip install -r requirements.txt
```

Запустіть сервер:

```bash
uvicorn main:app --reload
```

Backend буде доступний за адресою:

```text
http://localhost:8000
```

### 3. Запуск frontend

Відкрийте новий термінал і перейдіть у папку frontend:

```bash
cd frontend
```

Встановіть залежності:

```bash
npm install
```

Запустіть React-застосунок:

```bash
npm start
```

Frontend буде доступний за адресою:

```text
http://localhost:3000
```

## Змінні середовища

Для роботи email-підтвердження та відновлення пароля потрібно створити файл `.env` у папці backend.

Приклад:

```env
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your_email@gmail.com
SMTP_PASSWORD=your_app_password

APP_URL=http://localhost:8000
FRONTEND_URL=http://localhost:3000

SECRET_KEY=your_secret_key
```

> Не завантажуйте файл `.env` у GitHub. Додайте його до `.gitignore`.
